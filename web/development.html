Development Roadmap

From concept to release — follow our progress through each milestone.

🛠️ Foundation & UI March 2026
💬 Communication PoC April 2026
⚙️ Core LUA Engine May 2026
🌐 Web & Twitch Integration July 2026

🛠️

Phase 1: Foundation & UI

March 2026 — Active

Implementation Draft:

Core UI Architecture: Building a modular CSS framework from scratch using CSS Variables for theme consistency. The layout utilizes CSS Grid and Flexbox for responsiveness without heavy JS dependencies.

Translation Engine: Implementing a flat-file JSON translation system in `i18n.js` that allows instant language switching without page reloads by mapping `data-i18n` attributes to localized strings.

Tech Stack:

Vanilla CSS3, HTML5, i18next-lite

Checklist:

  • ✓ Establish global design system and color palette
  • ✓ Implement real-time translation engine (DE/EN)
  • ✓ Design interactive Hardcore Ruleset documentation
  • ✓ Create high-fidelity mockups for Character Profile cards
  • ✓ Build responsive landing page container
💬

Phase 2: Communication PoC

April 2026

Strategy:

Addon Syncing: Before building complex track logic, we must prove local addons can talk to each other synchronously.

Hidden Channels: Utilizing the `SendAddonMessage` API to silently broadcast active Rulesets and Death alerts across the Guild channel so all viewers stay in sync.

Tech Stack:

LUA (Blizzard API)

Checklist:

  • ○ Create `/tbchc ping` slash command
  • ○ Hook into `CHAT_MSG_ADDON` listen events
  • ○ Execute successful cross-client Ping/Pong test
⚙️

Phase 3: Core LUA Engine

May 2026

Death Tracking Matrix:

Authentic Engine: Implementing multi-step death verification combining `PLAYER_DEAD`, `UNIT_DIED`, and aura scanning to filter out false positives like Soulstones and Feign Death.

Saved Variables: Local configuration and death tallies persist across relogs securely using the Ace3 database framework.

Tech Stack:

LUA (Blizzard API), Ace3 Framework

Checklist:

  • ○ Build True-Death Verifier
  • ○ Trigger Hybrid Gear Wipes via `UNIT_INVENTORY_CHANGED`
  • ○ Build SSF Enforcer limits
🧪

Phase 4: Limited Beta

July 2026

Beta Strategy:

Closed Testing: First release to a small group of trusted guilds and streamers. Limited feature set — core death tracking, scoring, and basic GM tools only. No streamer integration yet.

Web Backend: A Node.js + PostgreSQL backend goes live for real-time death tickers, leaderboard calculations, and character data storage. WebSocket connections push death alerts to the website within milliseconds.

Tech Stack:

Node.js, Express, PostgreSQL, Socket.io, Twitch API v5

Checklist:

  • ○ Deploy backend API and database to production
  • ○ Invite top 10 HC guilds for closed beta
  • ○ Implement real-time WebSocket death ticker
  • ○ Test Twitch Extension penalty wheel integration
  • ○ Collect beta feedback and file bug reports
🚀

Phase 5: Full Release v1.0

August 2026

Scaling Strategy:

Global Rollout: Deploying the infrastructure across multiple Docker containers on AWS or a high-performance VPS.

Full Feature Set: All community-voted features enabled. Streamer tools, Twitch penalty wheel, Guild vs. Guild competitions (Team Tyler vs Team Soda style), and the full Companion App bridge go live.

High Availability: Utilizing Cloudflare's WAF to protect against the DDoS attacks common to large-scale private WoW projects, ensuring 24/7 leaderboard availability even during peak launches.

Tech Stack:

Docker, AWS ECS, Cloudflare WAF, Nginx, Electron (Companion App)

Checklist:

  • ○ Finalize production Docker Compose orchestration
  • ○ Execute stress test with 5k simulated character pushes
  • ○ Configure automated database backups & SSL
  • ○ Launch landing page marketing campaign
  • ○ Public release for all Hardcore Guilds
