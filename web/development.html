Master Project Plan

Technical Specifications & Implementation Milestones

🛠️

Phase 1: Foundation & UI

March 2026

Implementation Draft:

Core UI Architecture: Building a modular CSS framework from scratch using CSS Variables for theme consistency. The layout utilizes CSS Grid and Flexbox for responsiveness without heavy JS dependencies.

Translation Engine: Implementing a flat-file JSON translation system in `i18n.js` that allows instant language switching without page reloads by mapping `data-i18n` attributes to localized strings.

Tech Stack:

Vanilla CSS3, HTML5, i18next-lite

Checklist:

  • ✓ Establish global design system and color palette
  • ✓ Implement real-time translation engine (DE/EN)
  • ○ Design interactive Hardcore Ruleset documentation
  • ○ Create high-fidelity mockups for Character Profile cards
  • ○ Build responsive landing page container
📦

Phase 2: Addon Core

April 2026

Strategy:

Lua State Machine: The addon acts as a passive listener for Blizzard events. We will implement robust hooks into `COMBAT_LOG_EVENT_UNFILTERED` and `UNIT_DIED` to track deaths instantly.

Anti-Cheat: For HC+, we need to verify deaths. The addon will generate a unique hash for every level-up and death event, combining character names, timestamps, and server ID to prevent manual data spoofing.

Tech Stack:

Lua (Blizzard API), HMAC-SHA256, ACE3 Framework

Checklist:

  • ○ Engineer core death tracking logic in Lua
  • ○ Implement secure data encryption for event hashes
  • ○ Build in-game Hardcore status UI (Unit Frames)
  • ○ Develop localized quest & mechanic tooltips
  • ○ Create export-to-web data protocol
📊

Phase 3: Web Backend & DB

May 2026

Backend Infrastructure:

Real-time API: Using Node.js and Socket.io to push death alerts to the website within milliseconds. This allows for the "Global Death Ticker" feature.

Leaderboard Logic: A PostgreSQL database will store character data. Rankings are calculated based on a weighted formula: (Level * 100) + Quest Completion % - Death Penalties.

Tech Stack:

Node.js, Express, PostgreSQL, Socket.io

Checklist:

  • ○ Set up production-grade PostgreSQL schema
  • ○ Build authenticated REST API for Companion App communication
  • ○ Implement real-time WebSocket death ticker
  • ○ Design algorithmic ranking system for Leaderboards
  • ○ Develop character history viewing backend
🧪

Phase 4: Streamer Tools

June 2026

Streamer Ecosystem:

Chaos Integration: Building a Twitch Extension that allows viewers to spin a 'Penalty Gear'. If a penalty wins, the backend pushes a command to the streamer's Desktop Companion App.

Companion App: A specialized Electron application that bridges the web browser and the WoW Addon files, allowing automated UI updates and penalty execution (e.g., deleting an item).

Tech Stack:

Twitch API v5, Electron, OAuth2.0

Checklist:

  • ○ Build Electron Bridge App for file-system access
  • ○ Register official Twitch Extension on Dev Dashboard
  • ○ Develop interactive 'Penalty Gear' animation
  • ○ Engineer OBS Overlay templates for death tracking
  • ○ Test Twitch Bits integration for penalty tiering
🚀

Phase 5: Release v1.0

August 2026

Scaling Strategy:

Global Rollout: Deploying the infrastructure across multiple Docker containers on AWS or a high-performance VPS.

High Availability: Utilizing Cloudflare's WAF to protect against the DDoS attacks common to large-scale private WoW projects, ensuring 24/7 leaderboard availability even during peak launches.

Tech Stack:

Docker, AWS ECS, Cloudflare WAF, Nginx

Checklist:

  • ○ Finalize production Docker Compose orchestration
  • ○ Execute stress test with 5k simulated character pushes
  • ○ Configure automated database backups & SSL
  • ○ Launch landing page marketing campaign
  • ○ Public Beta release for top 10 Hardcore Guilds
