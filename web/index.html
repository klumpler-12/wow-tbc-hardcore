Four Ways to Play

From classic permadeath to competitive leagues — pick the mode that fits your playstyle. Each brings unique mechanics to TBC.

💀

Hardcore Classic

The original ruleset, just like Classic HC. One life — death means your character is permanently done.

⚰️ Character marked DEAD — guild banned, tracked on death board, voided forever
🔒 Full SSF enforced — no mail, no AH, no trading
🗡️ All gear and inventory destroyed on death
🚫 No boosts, no shortcuts — level 1 to 70 or die trying
Severity
⚡

Hardcore Plus

Classic HC rules evolved: unlock boosts after your first 70, and optionally toggle Gear Battle duels for extra stakes.

⚰️ Death = permanent void, same as Classic
🔓 Reach 70 once → unlock 58 boost for future characters
⚔️ Gear Battle (optional toggle) — challenge another player, each wagers one item. Winner gets score + title, loser's item is deleted
🏅 Gear Battle wins earn exclusive titles and leaderboard points
Severity
🔥

Softcore

You survive, but barely. Death strips away everything you've earned — rebuild from the ashes.

💔 All equipped gear, inventory, and gold destroyed
📦 Bank items and alts unaffected — rebuild from stash
🤝 Guild trading only — partial SSF with trusted allies
Severity
🏆

Nullcore League

The competitive format. No permadeath — pure score-based racing where every action earns or costs points.

📊 Deaths deduct points — quests, dungeons, achievements earn points
🏁 Season-based competition with time-limited leaderboards
🎖️ Tier rewards: Bronze → Silver → Gold → Gladiator
🌍 No trade restrictions — play normally, compete on merit
Severity
What's Under The Hood

Built for Outland

Everything you need for organized hardcore TBC, running entirely client-side via the WoW Addon API.

📡

Death Monitoring

Real-time combat log analysis detects deaths instantly — recording cause, zone, level, killer, and timestamp.

⚖️

Penalty Engine

Configurable penalty tiers from character voiding to gear deletion to score deductions. Your guild, your rules.

🏅

Scoring System

Points for quests, dungeons, achievements, and milestones. Deaths subtract. Compete on the leaderboard.

👥

Guild Integration

Auto-kick on HC death, death announcements, guild-wide sync, and a memorial archive for the fallen.

🔐

SSF Enforcement

Optional mail lock, trade restrictions, and AH bans. Enforced through addon verification and guild rules.

📊

Live Leaderboards

Real-time scores synced via addon messages across guild members. See who's leading the charge into Outland.

Streamer & Viewer Integration

Hardcore as a Spectator Sport

Turn every death, duel, and milestone into content. Real-time overlays, viewer interaction, and public profiles — all bridged through a companion app.

🎥

OBS Overlay API

Companion app exposes a local WebSocket that OBS Browser Sources connect to — showing live data directly on stream.

  • Live HP bar, mana, and combat status
  • Current score and leaderboard position
  • Death counter and active mode indicator
  • Equipped gear display with item links
  • Kill feed and recent achievement ticker
🗳️

Viewer Penalty Votes

Viewers choose the punishment. Twitch polls or chat commands let the audience decide penalty severity in real-time.

  • Streamer dies → poll triggers: "Gear wipe or score penalty?"
  • Vote results pushed via companion app → SavedVariables → addon
  • Configurable options per mode (HC Plus allows viewer choice, HC Classic overrides)
  • Twitch Extension or chat bot integration
  • Timeout window to prevent spam — one vote per death event
👤

Public Player Profiles

Opt-in web profiles showing your HC journey. Link your Twitch, share your plans, flex your stats.

  • Streamer link (Twitch / YouTube / Kick) displayed on profile
  • Current character, mode, level, and plan notes
  • Full death history with cause, zone, and timestamp
  • Account-wide stats: total score, chars played, boost usage
  • Privacy controls — choose what's visible
🔔

Stream Event Alerts

Auto-triggered visual and audio alerts for key moments — making every milestone and death cinematic.

  • Death alert: screen flash, sound effect, death recap overlay
  • Level milestone: celebration animation + score popup
  • Gear Battle: split-screen wager reveal + result
  • First kill / boss kill: custom badge animation
  • OBS scene auto-switch on death (e.g. swap to death cam)
🎮

Viewer Challenges

Viewers spend channel points to influence the streamer's HC run — adding stakes and interactivity.

  • Spend points to force a Gear Battle challenge
  • Place bounties on the streamer (other HC players can claim)
  • Vote on which zone the streamer must quest in next
  • Trigger "gauntlet mode" — extra death penalty for X minutes
  • Reward top contributors with in-stream shoutouts
📊

Account-Wide Stats

One page to see your entire HC legacy — every character, every death, every title earned.

  • All characters: active, voided, boosted — with class colors
  • Achievement timeline with ripple effects (boost unlocks, follow-up chars)
  • Gear Battle W/L record and title collection
  • Total playtime, average death level, PvP stats
  • Shareable link + embeddable widget for stream panels
⚠️ Viewer → game interactions require legal review against Blizzard addon policy and Twitch Extension guidelines. Companion app architecture avoids direct API calls from the addon.

Raw concepts and brainstorms being explored. These aren't final features — they're possibilities being shaped.

Mechanic Brainstorm

PvP Death Penalties

PvP deaths should have unique penalties — not just ban or delete. Make PvP engaging within HC rules.

  • Score deductions scaled by opponent's level
  • Temporary gear lock (can't upgrade for X minutes)
  • Bounty system — killing high-score players earns bounty
  • Separate PvP ELO leaderboard
  • Flagged zones with unique PvP death rules
Mechanic Concept

Score-Based Item Unlocks

Reaching scoring milestones unlocks access to higher gear tiers, adding a progression layer within HC.

  • "Must earn X points before equipping Epic items"
  • Addon monitors equip events and enforces restrictions
  • Creates natural gating without arbitrary locks
Mechanic Concept

Gear Battle Duels

Optional toggle in HC Plus — challenge another player, each wagers an item. Winner earns score + title.

  • Custom UI for selecting wagered item
  • Both players confirm before duel starts
  • Loser's item auto-deleted by addon
  • Exclusive titles for win streaks
Anti-Cheat Researched

Backend Verification Layer

Blizzard's Item Restoration lets players recover deleted items every 24h — a cheating vector. Companion app solves this.

  • Companion app reads SavedVariables, posts to backend
  • Inventory hash snapshots detect restored items
  • Two-step verification: app must run before WoW launch
  • Optional — only for tournaments needing integrity
Integration Planned

Discord Bot Integration

A bot that bridges the addon with Discord for real-time event management and notifications.

  • Death notifications posted to guild channel
  • Leaderboard updates on schedule
  • Slash commands for player stats lookup
  • Event scheduling and tournament management
Feature Concept

Desktop / Browser Setup

Configure everything from a web dashboard — no need to be in-game to manage addon settings.

  • Web UI for modes, rules, guild settings
  • REST API for addon ↔ backend ↔ Discord
  • Leaderboard viewer accessible from browser
  • Settings sync via companion app → SavedVariables
Social Brainstorm

Mixed Group Play

Can HC and non-HC players group together? Needs careful registration and verification layers.

  • Characters must be pre-registered before receiving mail/items
  • Guild/player-level setting for mixed group rules
  • Multi-layer verification (server + Blizzard + addon)
  • Prevents non-HC players from feeding items to HC players
Business Brainstorm

Monetization Concepts

Sustainable revenue without paywalling core HC functionality.

  • Unlock fee for custom modes (not HC Classic/Plus)
  • Premium streamer tools: overlays, viewer controls
  • Advanced stats dashboard and data exports
  • Must verify legality with Blizzard addon policy
Mechanic Concept

Anti-Boost Detection

Optional toggle to prevent power-leveling. Detects when a high-level player is carrying and penalizes accordingly.

  • Grouped with player 20+ levels higher who uses abilities near you
  • If you complete a quest under those conditions → quest auto-abandoned
  • XP from kills while boosted player is nearby could be zeroed or flagged
  • Configurable threshold (guild can set level gap tolerance)
  • Togglable per mode — strict for HC Classic, optional for HC Plus
