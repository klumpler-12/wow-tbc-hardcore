# Glossary

| Term | Definition |
|------|-----------|
| **HC** | Hardcore. Permadeath mode where death means character deletion or rollback. |
| **Hybrid HC** | The single game mode of TBC Hybrid Hardcore. HC rules during leveling, instance lives for endgame content. |
| **SSF** | Solo Self-Found. Restriction mode disabling trading, mail, and AH. |
| **Fraction** | A sub-guild team within a larger guild. Fractions compete against each other for points. |
| **GM** | Guild Master. Player with admin control over guild's TBC Hybrid Hardcore settings. |
| **MVA** | Minimum Viable Addon. Smallest feature set that is useful and testable. |
| **Ruleset** | Complete configuration of hard and soft rules defining Hybrid HC behavior for a guild. |
| **Soft Rule** | A rule that can be toggled or configured by the GM. |
| **Hard Rule** | An immutable rule defining core identity of the mode. |
| **Checkpoint** | Level milestone (default 58) that unlocks class-based rollback on death instead of full restart. |
| **Companion App** | Desktop app bridging WoW addon and web backend via SavedVariables file watching and HTTPS sync. |
| **SavedVariables** | WoW's persistence mechanism. Lua tables written to disk between sessions. |
| **HMAC-SHA256** | Cryptographic signing for data integrity between companion app and backend. |
| **AceAddon** | Popular WoW addon framework library for module management, events, and UI. |
| **CMaNGOS** | Open-source WoW server emulator used for development and testing. |
| **OF3** | OnlyFangs 3. Large-scale community HC event by Sodapoppin and NMP. |
| **BiS** | Best in Slot. Optimal gear piece for a specific slot and class/spec. |
| **Draft** | System where fraction leaders take turns selecting players for their roster. |
| **Micro-Achievement** | Small, specific accomplishment awarding modest points. Thousands of these exist. |
